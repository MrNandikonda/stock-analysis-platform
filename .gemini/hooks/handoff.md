# Handoff Hook Runbook

Use this checklist in the final response or when transferring work.

Include:
- What was changed.
- Exact files created or updated.
- Why the changes improve future Gemini work.
- Validation commands run and results.
- Checks not run and why.
- Residual risks or recommended next steps.

For code changes, also include:
- API/schema contract impacts.
- Migration impacts.
- Scheduler/Docker impacts.
- Frontend/backend alignment notes.
