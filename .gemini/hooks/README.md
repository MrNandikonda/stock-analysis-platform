# Gemini Hook Runbooks

This directory documents repeatable hook-style workflows for Gemini sessions. These are not executable hooks unless future tooling explicitly wires them up.

Use these runbooks to make agent behavior consistent:
- `pre_edit.md` - before touching files.
- `pre_commit.md` - before finalizing changes.
- `handoff.md` - final summary and transfer checklist.

The goal is predictable, safe, repo-native work rather than extra automation.
